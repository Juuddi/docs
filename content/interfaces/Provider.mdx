A **Provider** is the primary method to interact with the read-only
 content on Ethereum.

 It allows access to details about accounts, blocks and transactions
 and the ability to query event logs and simulate contract execution.

 Account data includes the [**balance**](/content/interfaces/Provider.mdx#getbalance),
 [**getTransactionCount**](/content/interfaces/Provider.mdx#gettransactioncount), [**code**](/content/interfaces/Provider.mdx#getcode) and
 [**state trie storage**](/content/interfaces/Provider.mdx#getstorage).

 Simulating execution can be used to [**call**](/content/interfaces/Provider.mdx#call),
 [**estimateGas**](/content/interfaces/Provider.mdx#estimategas) and
 [**get transaction result**](/content/interfaces/Provider.mdx#gettransactionresult).

 The [**broadcastTransaction**](/content/interfaces/Provider.mdx#broadcasttransaction) is the only method which allows updating
 the blockchain, but it is usually accessed by a [Signer](../interfaces/Signer), since a
 private key must be used to sign the transaction before it can be
 broadcast.

### waitForTransaction()

```ts
waitForTransaction(
   hash, 
   confirms?, 
timeout?): Promise<null | TransactionReceipt>
```

Waits until the transaction `hash` is mined and has `confirms`
 confirmations.

#### Parameters

| Parameter | Type | Description |
| :------ | :------ | :------ |
| `hash` | `string` | The transaction hash to wait for. |
| `confirms`? | `number` | The number of confirmations to wait for. |
| `timeout`? | `number` | The number of milliseconds to wait before timing out. |

#### Returns

`Promise`\<`null` \| [`TransactionReceipt`](/content/classes/TransactionReceipt.mdx)\>

A promise resolving to the transaction receipt or null if not found.
