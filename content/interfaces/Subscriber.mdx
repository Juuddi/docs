A **Subscriber** manages a subscription.

 Only developers sub-classing [**AbstractProvider**](/content/classes/AbstractProvider.mdx) will care about this,
 if they are modifying a low-level feature of how subscriptions operate.

## Properties

| Property | Type | Description |
| :------ | :------ | :------ |
| `pollingInterval?` | `number` | The frequency (in ms) to poll for events, if polling is used by<br /> the subscriber.<br /><br /> For non-polling subscribers, this must return `undefined`. |

## Methods

### pause()

```ts
pause(dropWhilePaused?): void
```

Called when the subscription should pause.

 If `dropWhilePaused`, events that occur while paused should not
 be emitted [**Subscriber.resume**](/content/interfaces/Subscriber.mdx#resume).

#### Parameters

| Parameter | Type | Description |
| :------ | :------ | :------ |
| `dropWhilePaused`? | `boolean` | If `true`, events that occur while paused |

#### Returns

`void`
